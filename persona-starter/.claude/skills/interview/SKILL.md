# インタビューモード — 分身を育てる

このスキルが呼ばれたら、優秀なインタビュアーとして甲斐 充さん本人に取材し、
回答を `persona/mitsuru.md` に蓄積してください。

## 準備

1. `persona/mitsuru.md` を読み、各セクションの充実度を把握する（「（未取材）」または文章量が少ないセクション＝薄い）
2. `interviews/` 内の過去ログを確認し、**既に聞いた質問を把握する**（同じ・類似の質問は絶対にしない）
3. 今日の日付のログファイル `interviews/YYYY-MM-DD.md` を用意する（無ければ `# インタビュー YYYY-MM-DD` の見出しで新規作成）

## 質問のルール

- **質問は必ず1つずつ。**まとめて複数聞かない
- 相手の直前の回答に軽く共感・相槌を打ってから次の質問に移る（1〜2文）
- 最も薄いセクションを優先的に掘り下げる
- 抽象的な質問より、**具体的なエピソードを引き出す質問**を好む（「〜だった経験はありますか？」「そのとき何を考えましたか？」）
- 敬語で、親しみやすく
- 回答を受け取るたびに、Q&Aをそのまま今日のログファイルに追記する:

```markdown
## Q: （質問文）
A: （回答をそのまま。要約せず本人の言い回しを残す）
```

## 蒸留（ペルソナ文書への反映）

**5問ごと**、またはユーザーが「ここまで」「終わり」と言ったら、たまったQ&Aを
`persona/mitsuru.md` の該当セクションに統合する。

- 既存の内容は保持しつつ、新情報を追記・統合する（矛盾があれば新しい回答を優先）
- 本人の言い回し・固有名詞・数字はできるだけ残す
- 各セクションは箇条書き中心の簡潔なmarkdown
- 回答内容だけでなく、**回答の文体から読み取れる口癖・語尾・話すリズムを「口調・話し方」セクションに必ず蓄積する**
- 反映が終わったら「今回追加した内容の一言要約」を伝える

## 終了時

セッションを終えるとき（蒸留後）に必ず変更をコミットしてプッシュする:

```bash
git add persona/ interviews/ && git commit -m "interview: ペルソナ更新" && git push -u origin main
```

このリポジトリはprivateなので、人格データを含めてコミットしてよい。
