# 🧠 時事クイズ Bot 管理 — 全体説明書

時事ニュースからクイズを **Claude APIで自動生成** し、**人間が承認してからXへ投稿する** 承認制Botの管理アプリです。
スマホ1台（Pixel 10a想定）で完結し、サーバー不要・単一HTMLのPWA（ホーム画面アプリ）として動作します。

```
ニュース検索 → クイズ自動生成 → 【あなたが承認】 → Xアプリで投稿 → 数時間後に正解を返信
```

- **サーバー不要** — `index.html` 1ファイルがアプリ本体。静的ホスティング（GitHub Pages等）に置くだけ
- **データは端末内のみ** — すべてブラウザのlocalStorageに保存。外部送信先はAnthropic API（クイズ生成）と、任意設定のPC連携Webhookのみ
- **承認制** — 無承認の自動投稿は行わない設計（Xの自動化ルールに配慮）

---

## 1. 初期セットアップ

1. **ホスティング** — このリポジトリを GitHub Pages 等の静的ホスティングで公開し、そのURLをChromeで開く
2. **PWAインストール** — Chromeメニュー（⋮）→「ホーム画面に追加」→「インストール」。以後はホーム画面の「クイズBot」アイコンから全画面で起動
3. **通知許可** — 初回起動時のダイアログで「許可」（リマインダー・速報の通知に必要）
4. **APIキー設定** — [console.anthropic.com](https://console.anthropic.com) → API Keys → Create Key で発行し、アプリの「🔑 APIキー設定」に保存

> 🔒 APIキーはこの端末のブラウザ内（localStorage）にのみ保存されます。クイズ生成時に api.anthropic.com へ直接送信される以外、外部には出ません。

## 2. 画面と機能一覧

アプリの画面上から順に、全カードの役割です。

| カード | 機能 |
|---|---|
| **ヘッダーのスイッチ** | メイン電源。ONにすると即1問生成し、以後は設定間隔（30分/1時間/2時間）で自動生成。安全のため起動時は常にOFF |
| **☁️ クラウドBot** | GitHub Actionsで動く自動投稿（[cloud-bot](cloud-bot/README.md)）の稼働状況を表示。今月の投稿数・本日の速報件数・正解返信待ち・投稿履歴（タップでXの投稿を開ける）。5分ごとに自動更新、オフライン時は直近の取得結果をキャッシュ表示 |
| **💰 X収益化トラッカー** | 投稿ペース（今日・直近7日・連続投稿日数）を承認履歴から自動集計。収益化3条件（フォロワー500人・直近3ヶ月インプレッション500万・Premium加入）の進捗バー。💴推定月間収益の試算（直近28日インプ×想定単価。単価は非公表のため自分で調整する非公式の目安） |
| **⏳ 承認待ち** | 生成されたクイズが並ぶ。承認するとXアプリへの共有画面が開き、却下すると破棄。承認期限（30分/1時間）を過ぎると自動破棄 |
| **⚡ その場で生成** | 手動生成。3モード: 🔍**テーマ**（キーワードからニュース検索して出題）/ 𝕏**引用ポスト**（ポストURLから引用リポスト用クイズ）/ 🔗**リンク記事**（記事URLから「読めば分かる」形式）。1問・3問・5問連続生成 |
| **💬 正解返信待ち** | 承認済みクイズの正解返信キュー。設定時間（1〜3時間）経過で通知が来て、出典リンク付きの正解をXで返信できる |
| **🔑 APIキー設定** | Anthropic APIキーの保存・削除 |
| **⚙️ 設定** | 生成間隔 / 承認期限 / 正解返信までの時間 / 出題ジャンル（時事全般・国際・国内・経済・科学技術・スポーツ） |
| **⏰ 定時リマインダー** | 7時・12時・21時（日本のXゴールデンタイム）に自動生成＋通知。個別ON/OFF |
| **🔴 速報ニュース監視** | 1時間ごとに速報を検索し、新着があれば即クイズ生成＋通知。海外ニュース優先度を設定可能。メインスイッチONが必要 |
| **📰 まとめ記事** | 生成済みクイズから週間（直近7日）・月間（直近30日）まとめを作成。X長文記事用のストック資産に |
| **🔗 PC連携** | PCの自動化システム（n8n / Make / 自作スクリプト等）との接続。承認時にJSONをWebhookへ自動送信、全データのエクスポート/インポート。→ 詳細は [docs/pc-integration.md](docs/pc-integration.md) |
| **📚 過去の問題** | 全生成履歴（最大200件保存）。キーワード検索・再投稿・コピー |
| **📜 アクティビティログ** | 直近の動作記録（最大50件） |

## 3. クイズのライフサイクル

```
生成（自動 / 手動 / 速報）
   │
   ▼
⏳ 承認待ち（pending）──── 却下 or 期限切れ ───→ 🗑 破棄
   │ 承認
   ▼
Xアプリで投稿（共有モーダルからコピー / X起動）
   │
   ├─ 正解返信ありモード → 💬 正解返信待ち（approved）→ 設定時間後に通知 → 正解＋出典を返信 → ✅ 完了（done）
   └─ 引用ポスト / リンク記事モード → 投稿すれば ✅ 完了（正解返信なし）
```

承認した瞬間、PC連携がONなら同じ内容がWebhookでPC側にも送信されます。

## 4. データの保存とプライバシー

| データ | 保存場所 | 上限 |
|---|---|---|
| クイズ履歴 | localStorage | 200件（古いものから削除） |
| アクティビティログ | localStorage | 50件 |
| PC連携の未送信キュー | localStorage | 100件 |
| APIキー / Webhookシークレット | localStorage | — |

- 「⬇️ エクスポート」で書き出すJSONに **APIキー・Webhook設定・シークレットは含まれません**
- 端末を手放す際は「🔑 APIキー設定 → 🗑 削除」を実行してください
- ブラウザのサイトデータを消去するとすべて消えます。定期的なエクスポートをおすすめします

## 5. 制約と注意事項

- **画面を開いている間のみ動作します。** バックグラウンドでの自動生成・監視・通知はありません（運用時間帯は画面スリープを延ばすのがおすすめ。→ [運用ガイド](docs/pixel10a-x-monetization-guide.md)）
- **投稿は必ず人間が承認します。** 無承認の全自動投稿に改造すると、Xの自動化ルールに抵触しアカウント凍結・収益化否認のリスクがあります
- クイズ生成にはAnthropic APIの利用料がかかります（モデル: Claude Sonnet）
- X収益化の条件・単価はXの規約変更で変わることがあります。最新情報はXアプリの「収益化」画面で確認してください

## 7. トラブルシューティング

| 症状 | 対処 |
|---|---|
| 更新が画面に反映されない | PWAのキャッシュが古い可能性。ページを数回再読み込み、直らなければアプリを削除して再インストール（データはlocalStorageに残ります） |
| 通知が来ない | 端末設定でChromeの通知許可と電池最適化の解除（設定→アプリ→Chrome→バッテリー→「制限なし」）を確認。アプリの画面が開いていることも必要 |
| クイズ生成に失敗する | APIキーが正しいか、Anthropicアカウントの残高・レート制限を確認。ログに失敗理由が出ます |
| PC連携の送信が失敗する | 失敗分は未送信キューに退避され、回線復帰時・起動時・5分ごとに自動再送されます（アプリの動作には影響なし）。「📤 未送信分を今すぐ再送」で手動再送も可能。恒常的に失敗する場合はURL・PC側サーバー・CORS設定を確認 |
